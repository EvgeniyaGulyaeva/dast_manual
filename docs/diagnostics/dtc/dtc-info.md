# Информация об ошибке

Используйте страницу DTC, чтобы получить всю доступную информацию о конкретном диагностическом коде неисправности, зарегистрированном в автомобиле.

Страница DTC является отправной точкой для анализа первопричины неисправности. Она предоставляет минимально необходимый, но достаточный набор данных, полученных непосредственно от автомобиля, и позволяет диагносту самостоятельно:

- понять условия возникновения ошибки;
- оценить влияние неисправности на работу автомобиля;
- проанализировать события, предшествовавшие ошибке;
- принять решение о дальнейших диагностических действиях.

В отличие от плана проверок (guided troubleshooting), страница DTC не подсказывает решение пошагово, а предоставляет фактические данные для аналитической работы. Без этой страницы невозможно корректно работать с DTC, анализировать жалобы клиента и устранять неисправности.

Cтраница DTC используется для следующих задач:
- просмотр детальной информации о коде ошибки, считанном с автомобиля;
- анализ Freeze Frame данных, зафиксированных в момент события;
- переход к руководству по диагностике и устранению неисправности;
- запуск пошагового плана проверок (если доступен);
- просмотр диагностических данных, сформированных DiagCore и связанных с данным DTC.

## Параметры на момент события (Freeze Frame)

Freeze Frame — это набор дополнительных параметров, сохраненных в момент фиксации ошибки. Эти данные играют вспомогательную, но критически важную роль при диагностике.

_как с ними работать? на что обращать внимание? есть ли рекомендации типа "для этого вида ошибок важно смотреть на эти параметры, для другого – на другие?_

Freeze frame-данные могут быть:
- глобальными — поддерживаются всеми ЭБУ, способными сохранять ошибки;
- локальными — специфичны для конкретного ЭБУ и поддерживаются не всегда.

Поддерживаемый набор параметров определяется диагностической спецификацией конкретного блока управления.

### План проверок

_Если для данного DTC доступен план проверок, на странице отображается соответствующая кнопка._

* Переход к плану проверок открывает пошаговый сценарий диагностики, сформированный DAST.
* _как выглядит план проверок? как с ним работать?_

Если план проверок отсутствует:
* используйте техническую документацию по ЭБУ;
* выполняйте диагностику вручную, опираясь на описание ошибки, условия её появления и Freeze Frame данные.

## Возможные проблемы и ограничения 

:::note
Не все электронные блоки управления поддерживают Freeze Frame и расширенные диагностические данные.
:::

В случае, если:

* информации недостаточно для анализа;
* план проверок отсутствует;
* неисправность не удаётся устранить самостоятельно,

создайте обращение в службу поддержки Atom и приложите экспортированные данные DTC и Freeze Frame.
